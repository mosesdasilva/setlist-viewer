## Author's notation

- Section Codes use the first letter of common Section names: `I` Intro, `V` Verse, `T` Tag, `P` Pre-Chorus, `C` Chorus, `B` Bridge, and `E` Ending.
- Each grid square is one Bar. Chart Rows commonly contain four Bars separated by vertical grid lines.
- A Slash Chord is stacked like a fraction; for example, `1` over `3` means `1/3`.
- Material written to the right of a Chart Row is a Row Note. It may be a performance instruction such as `Rakes` or a melody reference.
- Red writing indicates a Melody Passage. Parentheses group the passage, and internal vertical lines align melody fragments with Bars.
- A number enclosed by a diamond is a Diamond Chord; the Ending contains a diamond `3`.
- The chart writes the Expanded Arrangement in performance order instead of relying on navigation or repeat jumps.

## V1 questions exposed

These facts feed [Define the v1 Nashville chart language](https://github.com/mosesdasilva/setlist-viewer/issues/4):

- Whether melody references remain opaque Row Note text in v1, since melody-specific features are outside the map's scope.
- How the source notation encodes stacked Slash Chords and Diamond Chords while rendering familiar symbols.
- Whether Row Notes need an explicit distinction between Performance Directions and Melody Passages.
- How Section Codes map to full Section names without losing the author's compact vocabulary.
